# PRD：首页动态化、公开观察流与周更发布

## 问题

当前采集与公开站之间存在明显落差：仓库快照已有数千条 Signal，但公开站主要消费通过硬门禁的 Event。新增来源和新 Signal 不会自然改变首页，读者也难以感知系统正在持续观察什么。同时首页三个核心区域都是静态排序，重复刷新不会暴露更多判断入口。

## 目标

1. 在不降低事实发布门禁的前提下，让首页每次刷新暴露不同但仍然可信的内容入口。
2. 将尽可能多的采集结果通过隐私安全、许可克制的公开 DTO 呈现为“来源观察流”。
3. 把自动化节奏从日内高频收敛为每日一次数据刷新，并稳定产出一个可订阅、可回链的 GitHub 周报 Issue。
4. 完成 0.10.0 的版本、文档、仓库元数据、Release、Pages 与 Actions 闭环。

## 用户体验

- 用户刷新首页时，最新趋势判断在有证据的六个方向中随机出现一个。
- 近期变化使用 `Math.random()` 从最近 12 个符合条件的 Event 中随机抽取 6 个紧凑条目；桌面端一行两个，每条仍可打开证据抽屉。
- 最新趋势判断右上角提供“换一个”按钮，点击后随机切换到另一个有证据的趋势方向。
- 六个行业趋势块全部同时显示；每个块内部轮播该趋势下的多个最新 Event，支持自动播放、上一项、下一项、圆点定位、键盘和触摸滚动。
- 六条趋势使用六种稳定主题色；主题色贯穿首页行业趋势、趋势判断详情和阶段轨迹。
- 阶段轨迹使用有间距的独立阶段卡、序号节点与连接箭头，明确表达阶段之间的推进和变化。
- 趋势切换使用一次轻量上浮淡入；主要页面模块进入视口时执行一次滚入动效，并尊重系统的减少动态效果设置。近期变化只在刷新页面时重新随机，不展示单独换批按钮。
- 用户可进入“来源动态”页面，以瀑布流浏览、搜索和筛选公开安全信号，阅读限长简述并继续访问原始来源。
- 页面说明未完成收敛的条目不会替代 Event，但不使用大块告警卡干扰信息消费。

## 非目标

- 不把所有 Signal 自动转成 Event。
- 不公开 raw payload、内部 triage、数据库 ID、管理字段、token、cookie 或私有备注。
- 不绕过登录、WAF、robots、许可、CAPTCHA 或 SSRF 门禁。
- 不为“动态感”制造虚假热度、虚假随机数据或自动滚动陷阱。

## 验收标准

- 首页至少提供 2 个可选趋势判断、8 个近期 Event 候选、6 个同时可见的行业趋势块，并让每块拥有多个 Event 轮播项。
- 重新加载时使用浏览器端随机选择；无 JavaScript 时仍显示第一组完整内容。
- 公开观察 DTO 的每条记录只包含标题、限长纯文本描述、原文 URL、来源、时间、分类、标签和语言等 allowlist 字段。
- 观察页能呈现当前快照中的全部安全 Signal，并支持增量显示与搜索筛选。
- Data Refresh 使用每日 cron；Source Audit、Monitor 与 Quality Guard 使用每周 cron，不再存在逐小时高频 cron。
- 每日 Data Refresh 成功后触发 Pages；只有周日计划任务或显式手动运行才幂等写入当前 ISO 周的 `weekly-brief` Issue。
- 周报正文先给一句话判断，同一 Event 只展开一次；关键变化和行动建议各不超过 3 条，空趋势合并表达，运行指标默认折叠。
- `npm run check`、`npm run build`、关键浏览器验收、CI、Release、Pages 与线上 HTTP/内容检查全部通过。
