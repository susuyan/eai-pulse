# Changelog

所有值得用户感知的 Agent Pulse 变化都会记录在这里。版本遵循语义化版本，能力状态分为 planned、experimental 和 operational；只有拥有代码、测试或运行证据的能力才会进入 release。

## [Unreleased]

### 开发中

- 趋势详情的阶段证据不再等待滚动触发动画，页面首次渲染时直接展示；其他非证据模块继续保留滚入反馈。
- 修复行动参考 tabs 容器误产生纵向滚动的问题；明确关闭纵向溢出，仅在极窄屏保留横向容错，并隐藏滚动条。
- GitHub 周报改为一句话判断、关键变化、三项行动和仍需验证的紧凑结构；同一 Event 不再按趋势重复展开，空趋势合并为一行，运行指标收进折叠区。

## [0.10.0] - 2026-07-14

### 厂商与来源覆盖

- 事件脉络从按品牌补漏升级为 22 家重点厂商覆盖矩阵；新增 27 个 shadow-first 官方来源和 32 个可检索关键 Event，覆盖全球前沿模型、中国重点模型团队与 AI 原生搜索产品，并以契约测试阻止“有来源、无事件”的覆盖退化。
- 最终 411 源全量审计得到 261 healthy、27 degraded、71 failed、52 skipped，395 个可达、366 个成功拉取、272 个解析出内容。
- 全来源回填在 411 个目录来源中实际执行 284 个、按生命周期或采集边界跳过 127 个；拉取 1,609 条、创建 299 条规范化 Signal、去重 1,310 条，公开观察流累计 4,386 条，16 个来源错误均被单源隔离且未中断整批任务。

### 静态站性能

- 静态导出改为压缩 CSS、JavaScript 与 JSON，主脚本提前进入浏览器下载队列；全量来源动态只在搜索、切换地域或加载更多时请求，GitHub Stars 与第三方行情组件延后到空闲或接近视口时加载。
- 来源动态的地域下拉改为统一的自定义控件样式，同时保留键盘访问与原生选择控件的可访问性。
- Data Refresh 改为北京时间每天 20:17 抓取并刷新 Pages，使站点数据与生成日期尽量保持当天最新；`weekly-brief` Issue 仍只在周日或显式触发时按 ISO 周幂等更新。

### 趋势判断交互

- 取消全局“展示模式”和简洁 / 全量切换，趋势判断默认采用简洁呈现；阶段轨迹、阶段证据、角色判断和观察源池分别在模块内部按需展开。
- 阶段轨迹的展开 / 收起控件位于模块标题栏右侧，并一次控制整条轨迹；阶段证据使用与轨迹一致的 `01`、`02` 序号映射，帮助读者对应阶段。
- 阶段轨迹继续保持单行、阶段间距与居中连接箭头；展开状态只存在于当前页面，不跨趋势页持久化。

## [0.9.0] - 2026-07-13

### 更真实、更高密度的趋势判断

- 首页移除冗长产品宣言与阅读路径，首屏直接进入 `LATEST MATERIAL SHIFT`；全站内容区标题取消右上角说明文字，并系统性缩短首页、趋势、事件、来源和工具页标题。
- 趋势总览改为“六个视角”，总览与详情页导航统一为轻量横向列表，不再显示“选择 / 切换趋势”或复杂卡片元数据。
- 趋势详情移除 Event、Evidence、Source 等首屏统计仪表，阶段证据保留解读、下一信号与可回链事实；主导航页标题、间距和行动参考布局统一。
- 六条趋势不再统一压成三个模板阶段，改为各自 5—8 个真实演进阶段；阶段轨迹保持单行并支持触摸、键盘横向滚动。
- 关键证据按阶段展示阶段解读、Event、Evidence、独立事实信源与下一验证信号；全量模式取消旧版每条趋势最多 7 个节点的上限。
- 增量证据按发布日期进入对应阶段，同时保留原 Event 的事实起点和唯一回链；进行中阶段采用开放区间，未来更新可自动归组且不会污染旧阶段计数。
- 新增简洁/全量双密度模式并跨趋势页记忆选择；无 JavaScript 时仍保留完整公开证据 HTML。
- 四种角色改为 24 组人工审阅的决策镜头，包含核心回答、影响链、建议动作、继续观察、失效线索和 Event 回链。
- 趋势页同时展示相关观察源池，并明确声明 Source Catalog 只用于持续发现，不等于已经形成事实证据；低覆盖主线继续显示缺口。
- 紫色实心按钮统一改为白色文字；行动参考内容区移除 `.section.shell` 的纵向 padding，提高首屏信息密度。
- 独立订阅大区块移除，Watch 与 AI 周报以轻量链接并入 Footer；品牌副标题改为“AI 行业判断系统”。
- 首页新增“看清 AI 行业的关键变化”产品 Hero，以轻量证据网络表达动态；首个内容模块从单事件新闻卡改为趋势阶段、当前判断、变化原因、最新证据与下一信号。
- 首页移除“形成判断 / 继续深入”模块，相关页面入口并入 Footer；行动参考只展示“行动参考 / 关键角色 / 模型成本”三个入口，关键角色移除重复提示并等高。
- 阶段证据在简洁模式下支持逐阶段查看全部证据和收起，不必切换整页全量模式。
- 趋势详情移除难以解释的“证据缺口”块；Footer 重构为品牌、订阅、分组导航和元信息四层结构。
- 趋势总览移除低信息量的“理解框架”；Timeline 左侧粘性时间标记随滚动同时显示当前年月。
- 首页、趋势判断、事件脉络和行动参考分别使用证据网络、轨道、时间扫描与行动路径动效；语言切换移入暖灰渐变 Footer。
- 趋势判断只展示六个判断视角，默认进入“模型能力与研究”；“行业演化”拆为 Footer 入口下的独立页面，趋势详情不再展示中国实践模块。
- 全站字体切换为浏览器原生系统字体栈；链接焦点使用底部状态反馈，不再出现 outline 外框。
- 首页移除“本周研究”，行业趋势卡升级为整卡可点击并强化悬停与箭头反馈；事件脉络筛选项选中后自动滑动到横向容器中部。
- Footer 改为浅暖灰渐变、扩大品牌与留白，并以轻量轨道装饰替代整块深色背景；“单一官方资料”等内部化描述改为“官方来源 / 多源核验”，已显示信源数量时不再重复标签。
- 修复右上角 GitHub Stars 在构建元数据暂时不可用或窄屏下不展示的问题；缺值时由浏览器限时补取并缓存，移动端保留紧凑的图标与数量。

### 事件驱动判断与 GitHub 周报

- 全站取消“每天 10 分钟”和固定时长阅读承诺，改为持续监测，只在热点、重要产品更新或技术迭代带来新证据时更新判断。
- 移除 `daily-brief` renderer、Issue 写入与订阅入口；公开订阅只保留 Watch 和 `weekly-brief` 周报。
- 周报按 Asia/Shanghai ISO 周汇总六条趋势、研究、行动和覆盖状态；没有足以改变判断的新证据时明确保留原判断，不为频率制造趋势。
- Data Refresh、Source Audit、Quality Guard 与 Monitor 统一收敛为每周计划；使用忽略运行时间戳的公开内容指纹判断语义变化，Pages 只在公开内容实质变化时部署。
- Pages 的普通 push 触发范围收紧到站点代码、依赖与公开快照，减少文档等非站点变化造成的空部署。

### 动态首页与公开来源观察

- 首页“最新趋势判断”在有公开证据的多个趋势方向中按刷新随机展示；近期变化从最新 12 个合格 Event 中随机展示 6 个。
- 近期变化改为桌面端一行两条的 2×3 信息网格；六条趋势以六种主题色贯穿首页、趋势判断和分段阶段轨迹。
- 趋势切换与主要模块滚入采用克制的上浮淡入，并兼容 `prefers-reduced-motion`；近期变化只在刷新页面时重新随机，不额外展示换批按钮。
- 六个行业趋势块保持同时可见，每块内部轮播多个最新 Event，并支持自动播放、手动切换、键盘、触摸与 reduced-motion。
- 新增“来源动态”公开观察流，把 Signal 的标题、简述、来源、时间、分类、标签与原文链接通过 allowlist DTO 以瀑布流呈现；当前快照中的合规 Signal 可以在前台搜索、筛选和分批浏览。
- 来源动态与事件脉络使用彼此不同的右上角动效；页面不再展示大块事实边界告警卡，同时观察层仍不能绕过 Event 发布门禁。

### 更丰富的一手来源供给

- 新增独立的 100-source Wave 2 清单：80 个官方 GitHub Release Feed 与 20 个研究、产业、政策和资本 RSS/Atom，覆盖 Agent/协议、模型工程、评测、基础设施、中国开源生态与机器人。
- 新来源全部保持 disabled + shadow + candidate；实时 source audit、E3 隔离采集和 E4 投产继续使用不同门禁，不把目录登记描述成有效观测。
- 来源页新增领域、地域、采集通道和真实运行状态四维组合视图，并为健康、RSS/Atom、GitHub 与中国来源增加快速筛选。
- 新增池全量实测 100/100 healthy、95 effective、86 个满足 90 天新鲜度；86 个 E3 来源隔离采集新增 939 条 Signal，0 个采集错误，未直接发布 Event。
- 来源目录新增 18 个 Tier 1 宏观、央行、监管披露、交易所公告与授权市场数据入口。
- FRED、ALFRED、IMF WEO、BIS、BLS、BEA、美国财政部、ECB、OECD、World Bank 和 Eurostat 作为候选来源进入 shadow。
- 国家统计局、人民银行、CNINFO、SSE、SZSE 保持 manual/shadow；Nasdaq Data Link 保持 restricted，不绕过许可与访问边界。
- 所有新增来源默认关闭，不改变既有采集、排名、发布与页面结构。

### 可恢复的全来源同步

- 仓库快照改为单调 merge：较旧快照不会覆盖本地较新来源状态、更完整 Signal 内容或已发布/人工 Event，并完整保留来源运行历史与事件关系。
- 同一 canonical URL 继续只生成一个 Signal，但不同来源和重复抓取会写入独立 observation，保留首次、最近观测时间与累计次数。
- 新增 `collect --scope=all --backfill --drain`，完整枚举来源目录、拉取所有合规且具备 adapter 的来源，并对 manual、restricted、proposal 和隔离状态给出明确跳过原因。
- 数据刷新与来源审计在写快照前合并 `origin/main` 最新快照，避免长任务用旧基线覆盖远端增长；全程不使用 force push。
- 发布前 384 源全量审计得到 243 healthy、24 degraded、65 failed、52 skipped。
- 全来源回填实际执行 260 个、按生命周期或采集边界跳过 124 个；新增 432 条规范化 Signal，公开观察流累计 4,057 条，12 个来源错误均被单源隔离且未中断整批任务。

## [0.8.1] - 2026-07-13

### 更紧凑的认知入口

- 首页把“30 秒、3 分钟、10 分钟”阅读路径移到价值标题右侧并缩小，在窄屏继续使用单列布局。
- 趋势判断用六条可点击主线替换事件数、时间跨度和引导标签，让用户从标题下直接进入感兴趣的行业方向。
- 收紧星探机会标题、观察与假设之间的纵向间距，提高同屏信息密度。

### 更充足的星探机会

- 星探从创业、内容、工作三类扩展到创业、内容、工作、学习、产物、影响力六类，并优先补齐缺失类型。
- 系统持续维护至少 18 条按类型与标题去重的公开机会；生成内容 14 天后自动轮换，避免机会池长期不变或无限累积。
- 每日数据刷新单轮最多生成 12 条候选，让机会池在数据恢复或过期轮换后及时补足。

## [0.8.0] - 2026-07-13

### 从事件列表到产业演化地图

- 趋势判断重新收敛为模型能力与研究、Agent 与软件重构、产品与商业验证、基础设施与成本、资本与公司演化、全球创新版图六个互斥度更高的方向；每张卡片增加明确的详情入口。
- 时间线从 2022 年 Stable Diffusion 与 ChatGPT 时刻开始，新增 13 个 2022—2023 官方来源里程碑，并用六个阶段解释能力、产品、商业和产业结构如何演化。
- 阶段卡片同时展示代表项目的持续发展、战略转向、收购与停止状态，只使用可点击的官方资料，不把项目转型简化成“成功或死亡”。
- 阶段轨迹使用箭头连接，区域视角统一为“全球创新中的中国实践”，不再使用预设单向落后的追赶叙事。

### 更完整的事件与行动信息

- 移除事件脉络中的“论文批次状态”和“近三月密度”运营模块，筛选改为“全部变化、官方发布、论文与研究”及六个趋势问题。
- 证据抽屉扩展到 780px，呈现完整发展节点、背景、技术变化、行业影响、决策含义、置信度、下一观察和全部原始资料；移动端改为单列底部抽屉。
- 修复 reading journey、事件脉络底部、抽屉和趋势阶段的宽度与间距；星探页顶部状态与机会标签统一左对齐，空标签不再占位。
- 公开星探按类型与标准化标题确定性去重，并补充触发变化、目标用户、置信度、证据强度、新颖度、行动价值和完整实验建议。

### 每日情报发布

- 数据刷新工作流每 4 小时生成北京时间当天的 AI 日报，并按日期 marker 幂等创建或更新同一个 `daily-brief` GitHub Issue。
- 日报只读取公开静态 DTO，汇总当天变化、论文、行动参考、评测与证据覆盖，不暴露数据库、原始 payload 或管理字段。
- 新增历史覆盖、项目状态、星探去重、日报安全渲染和 Actions 幂等契约测试。

## [0.7.0] - 2026-07-13

### 产品体验重构

- 首页从模块目录重构为“30 秒知道变化、3 分钟理解影响、10 分钟形成判断”的完整消费路径，主导航改为今日必读、趋势判断、事件脉络和行动参考。
- Event 继续作为唯一事实节点；同一事件的首次出现、官方更新、外部讨论和行业反馈按时间聚合，事件列表默认按最近进展倒序。
- 事件详情统一回答发生了什么、改变了什么、为什么重要、影响谁、当前判断和下一观察，减少 Timeline、Evidence、Decision Tools 等系统语言。
- 来源页新增 Claude Code、OpenAI/Codex、DeepMind、Cursor、Windsurf、Lovable、Vercel AI、Cloudflare AI、MCP、A2A、Browser Use、AI Coding、AI Infra 和 AI Agent 覆盖体检，明确区分目录收录、最近健康与有效覆盖。
- 新增 Lovable Changelog RSS 与 A2A Protocol Releases 两个 shadow-first 第一方来源，并提供成功 fixture、失败契约复用与真实 endpoint 验证。

### 事件脉络与研究深度

- 事件脉络改为按年、月组织的倒序列表；证据详情统一通过桌面右侧、移动端底部抽屉打开，并支持可复制 URL、Esc、遮罩关闭和焦点回退。
- 顶部 GitHub 入口改为通用的 Star 分段组件，静态构建优先使用 24 小时内元数据，否则从 GitHub 官方仓库 API 获取真实 Star 数量。
- 修复研究来源长期停在 deferred 的事件性门槛：具有明确方法、评测对象和决策相关性的论文可进入 review，普通论文列表和短摘要仍保持隔离。
- 研究事件新增方法、行业影响和后续验证深度门禁；首页增加“本周值得读的技术论文”，事件页明确标记预印本与独立复现边界。
- 首批补入长上下文难度、多智能体协作、合成 Persona、多模态盲区、推理泄密和因果数据 Agent 六个研究事件，均回链 arXiv 原始页面。
- 论文展示改为“时间轴完整可见、首页高质量精选”双层门槛；研究筛选覆盖全部公开论文，同日 4 篇以上聚合为可展开的论文日报，研究候选摘要与分析深度门槛适度下调。
- 事件脉络标题缩短为“一个事件，一条完整脉络”，通过响应式字号在桌面与 390px 窄屏保持单行。
- 证据抽屉升级为全站按需组件：首页、趋势、研究、事件脉络、相关事件和星探入口均可原地打开；移除 Timeline 中 50 份重复详情预渲染，并修复空月份标题残留。

### 自主质量运营

- Web Scraper 在 HTML 卡片缺少可信日期时优先使用页面声明的 RSS/Atom；四个 arXiv 分类源改用官方 export Atom 查询，逐源复验均返回 50 条健康结果。
- 来源审计后自动协调生命周期：连续 2 次异常的生产源降级，连续 5 次失败的来源隔离，隔离源连续 3 次健康后回到 shadow；激活统一要求 20 次健康检查和 7 天观察。
- 来源雷达自动匹配 14 条已有一手源，并把 27 条没有一手 URL 的纯聚合线索归类为“身份信息不足”，真实待判定由 50 降至 9。
- 星探生成器改为越过冷却事件继续扫描完整候选池，优先选择最近 90 天的高价值事件；达到总分、证据、置信度和新颖度门禁的建议直接上架，其余自动归档。
- 后台来源按钮新增统一 readiness、中文阻塞原因、重复提交保护和操作后刷新；启用、隔离、单源拉取和观察模式使用同一服务端判定。
