# PRD：让趋势、事件与行动形成完整认知路径

## 1. 当前问题

1. Timeline 顶部的“论文批次状态”和“近三月密度”属于系统运营信息，占用了用户理解事件的首屏空间。
2. 现有六条主线中“技术演进”和“AGI 进展”高度重叠；“模型经济学”过窄；“中国追赶”把多元创新错误表达为单向落后关系。
3. 趋势卡片虽然整体可点击，但缺少明确按钮；阶段卡片彼此割裂，不能一眼看出阶段传导关系。
4. 事件抽屉宽度、信息层级和 Timeline 行布局不足，事实、背景、技术变化、影响、判断、下一观察没有完整呈现。
5. Timeline 筛选仍使用“一手证据”等系统术语，用户不知道筛选后获得什么。
6. 行动参考存在同一事件、同一类型的重复卡片；标签右对齐且在窄宽度被截断；公开页没有形成足够的信息密度。
7. 当前公开历史从 2024 年开始，缺少 2022 年生成式 AI 产品化起点与 2023 年模型、Copilot、开放生态和治理扩散。
8. 产品只展示成功节点，没有表达创业项目被收购、停止独立运营、关闭产品或转向其他商业路径。
9. 每日增量只进入快照和 Pages，没有形成便于订阅、分享和回看的 GitHub Issues 日报。

## 2. 趋势判断重组

保留六个一级方向，但按不同决策问题重新定义：

| 方向 | 回答的问题 | 由现有维度演化 |
| --- | --- | --- |
| 模型能力与研究 | 模型和研究范式发生了什么实质变化？ | 技术演进收敛模型/研究事实 |
| Agent 与软件重构 | AI 如何从回答问题进入执行任务和重写软件？ | AGI 进展改为可观察的软件行动 |
| 产品与商业验证 | 哪些产品形成采用、留存、收入和工作流控制点？ | 商业化扩展产品验证 |
| 基础设施与成本 | 算力、训练、推理、开源和单位任务成本如何变化？ | 模型经济学扩展为完整供给层 |
| 资本与公司演化 | 资金、并购、团队迁移和创业项目生死意味着什么？ | 投资视角加入公司生命周期 |
| 全球创新版图 | 中国、美国、欧洲与开放生态各自创造了什么？ | 替代“中国追赶”，不预设单向领先关系 |

“中国进展与全球坐标”统一改为“全球创新中的中国实践”，只做同维度事实比较，不使用追赶叙事。

## 3. 2022—今天的行业弧线

公开 Timeline 至少从 2022 年 8 月 Stable Diffusion 和 2022 年 11 月 ChatGPT 开始，形成以下阶段：

1. 2022 H2：生成式 AI 从实验室进入公众产品；
2. 2023 H1：GPT-4、Copilot、插件和企业入口扩散；
3. 2023 H2：开放模型、多模态与全球治理框架成形；
4. 2024：原生多模态、长上下文、推理时计算和早期 Agent；
5. 2025：Agent 协议、编码 Agent、低成本推理与产业重组；
6. 2026：长任务、真实工作、科学应用与平台集中。

每个阶段展示代表模型、产品、创业项目、基础设施、治理变化和公司生命周期。项目状态只允许使用官方公告或仍可核验的产品状态，区分 `active`、`pivoted`、`acquired`、`sunset`，不把“热度下降”伪装成公司死亡。

## 4. 页面体验

- Timeline 首屏直接进入搜索和用户价值筛选，不显示运营密度或论文批次模块。
- 筛选改为“全部变化”“官方发布”“论文研究”及六条趋势；“官方发布”明确表示只看公司、实验室或监管机构的原始公告。
- 趋势目录卡和首页趋势卡都提供可见的“查看趋势详情”按钮。
- 阶段轨迹使用箭头连接，桌面横向、移动端纵向，展示阶段之间的演进关系。
- 抽屉桌面宽度提升到 720–780px，分为事实摘要、事件发展、变化解读、决策影响、下一观察、原始证据和完整页面入口。
- 抽屉 Timeline 不再压缩成两列，日期、节点类型、标题、来源和摘要各自拥有稳定位置；默认展示全部去重证据。
- 统一 reading journey、section、Timeline 底部、抽屉和行动参考的宽度与垂直间距。
- 行动参考按归一化标题与证据事件去重；标签左对齐、允许换行；卡片显示目标用户、置信度、证据分、新颖度、为什么现在、最小动作、产物和反向信号。

## 5. 每日 GitHub Issue

- 每次 Data Refresh 完成静态导出后生成当日（Asia/Shanghai）日报。
- 同一天只维护一个 Issue，后续刷新更新原 Issue；不同日期创建新 Issue。
- 日报只消费公开 allowlist 数据，包含当日关键事件、研究、星探机会、系统增量和站点入口，不暴露数据库或原始 payload。
- 无新增公开事件时如实说明，并保留采集/公开数量摘要。

## 6. 验收

- Timeline 页面不再包含两个运营模块。
- 全站不出现“中国追赶”或 `China catch-up` 的用户可见文案。
- 六个趋势维度、按钮和详情页一致；阶段轨迹有箭头关系。
- 公开 Timeline 覆盖 2022、2023、2024、2025、2026。
- 至少展示 5 个有官方证据的项目转型、收购或终止节点。
- 抽屉信息完整、桌面和 390px 均无溢出；底部留白与全站 spacing token 一致。
- 星探去重后无重复标题，所有通过公开门禁的高置信内容均可见。
- Data Refresh 能创建/更新每日 Issue，marker、日期、标签和公开链接可验证。

## 追加需求：首屏层级与星探供给（v0.8.1）

- 首页的 30 秒、3 分钟、10 分钟阅读路径在桌面端缩小并放到价值标题右侧；窄屏恢复上下排列。
- 趋势总览用六个趋势入口替换“公开事件数 / 时间跨度 / 选择主线”状态标签，直接放在标题说明下方。
- 星探卡片收紧标题与正文间距，不牺牲标签、证据和行动信息。
- 星探公开供给覆盖创业、内容、工作、学习、产物、影响力六类，自动维持至少 18 条通过既有质量门禁且互不重复的公开机会。
- 达到公开池水位后停止无界生成；过期自动生成卡归档后再补位，seed 长期基线不因无过期时间被清理。
